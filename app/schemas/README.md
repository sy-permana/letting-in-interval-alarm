# Room Database Schemas

This directory contains exported Room database schemas for version control and migration testing.

## Purpose

- **Version Control**: Track database schema changes over time
- **Migration Testing**: Validate migrations between schema versions
- **Documentation**: Provide a reference for database structure

## Schema Files

Each schema file represents a specific database version:
- `1.json` - Initial database schema (version 1)
- `2.json` - Database schema version 2 (when created)
- etc.

## Usage

These schema files are automatically generated by Room when you build the project.
They should be committed to version control to track database changes.

## Migration Testing

Use these schemas with Room's `MigrationTestHelper` to test database migrations:

```kotlin
@RunWith(AndroidJUnit4::class)
class MigrationTest {
    @get:Rule
    val helper: MigrationTestHelper = MigrationTestHelper(
        InstrumentationRegistry.getInstrumentation(),
        AppDatabase::class.java.canonicalName,
        FrameworkSQLiteOpenHelperFactory()
    )

    @Test
    fun migrate1To2() {
        // Create database with version 1 schema
        helper.createDatabase(TEST_DB, 1).apply {
            // Insert test data
            close()
        }

        // Run migration and validate
        helper.runMigrationsAndValidate(TEST_DB, 2, true, MIGRATION_1_2)
    }
}
```

## Best Practices

1. **Never delete schema files** - They are needed for migration testing
2. **Review schema changes** - Check diffs when schemas change
3. **Test migrations** - Always test migrations before release
4. **Document breaking changes** - Note any data loss or transformations
